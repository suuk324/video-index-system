# CODE_REVIEW.md — 代码自检规则

## 使用场景

每次代码修改完成后，必须按本清单逐项自检，确认修改质量。

## 自检清单

### 一、修改范围检查

- [ ] 本次修改只涉及本次任务要求的内容
- [ ] 没有做无关的"顺手"改动
- [ ] 没有删除任何已有功能
- [ ] 没有改变任何已有 API 的请求/响应格式
- [ ] 没有改变任何已有数据库表结构

### 二、功能完整性检查

- [ ] 新增的代码能正常运行（无语法错误、无导入错误）
- [ ] 所有相关文件都已同步修改（例如改了路由，对应的前端调用也要检查）
- [ ] 错误处理完善（网络请求、数据库操作、文件读写都有异常处理）
- [ ] 边界情况处理（空值、空列表、超长字符串、特殊字符）

### 三、兼容性检查

- [ ] 新代码不会破坏已有 API 接口
- [ ] 新代码不会破坏前端已有功能
- [ ] 新增依赖不会与已有依赖冲突
- [ ] 数据库变更兼容已有数据（如有变更）

### 四、代码质量检查

- [ ] 变量命名清晰，能从名字理解用途
- [ ] 函数职责单一，不过长（建议不超过 50 行）
- [ ] 关键逻辑有注释说明
- [ ] 没有硬编码的魔法数字/字符串（应使用常量或配置）
- [ ] 没有未使用的导入、变量、函数

### 五、安全性检查

- [ ] 用户输入有验证和过滤
- [ ] SQL 查询使用参数化，无注入风险
- [ ] 文件路径有安全处理，无路径遍历风险
- [ ] 没有在日志中暴露敏感信息

## 自检结果

自检完成后，确认所有检查项通过。如有未通过项，必须先修复再提交。

## 例外

以下情况可简化检查：
- 纯文档修改：只需检查内容准确性
- 注释/格式调整：只需检查不影响代码逻辑
- 配置文件修改：只需检查格式正确
